### Context
n8n 사용 시, Publish 기능을 통해 워크플로우를 배포할 때 중복 실행되는 현상이 발생할 수 있습니다. 이는 Unpublish 시 트리거 연결이 즉시 해제되지 않아 중복 연결로 작동하기 때문입니다.

### Core
n8n에서 워크플로우 수정 및 배포 시 다음 순서를 권장합니다:
* 워크플로우 토글을 OFF 상태로 설정합니다.
* 최소 15초 이상 대기합니다.
* 필요한 노드 수정을 실시합니다.
* 다시 토글을 ON으로 전환합니다.

실행 중복 현상이 계속되면 다음 명령어를 통해 도커를 재시작할 수 있습니다:
```bash
cd ~/n8n && docker-compose restart
```

### Insight
워크플로우 개발은 로컬 환경에서 완료한 후 서버에 임포트하는 것이 좋습니다. 또한, 로컬과 서버의 n8n 버전을 일치시켜 버전에 따른 버그를 예방하고, 서버에서 직접 수정할 경우 도커를 통해 서비스를 재시작하는 것이 중요합니다.
