### 문맥
n8n에서 구글 드라이브 노드를 사용하여 바이너리 파일을 업로드하던 중, 파일명이 'undefined'로 저장되는 문제가 발생했습니다. 파일 내용은 정상적으로 업로드되지만, 파일명이 제대로 지정되지 않아 파일 관리에 어려움이 있었습니다.

### 핵심
구글 드라이브 노드 설정 시, 'Property Name'을 이전 노드에서 정의한 바이너리 키값과 정확히 일치시켜야 합니다. 예시:

```json
{
  "previousNodeData": {
    "json": {},
    "binary": {
      "data": {
        // 바이너리 데이터
      }
    }
  }
}
```

구글 드라이브 노드 설정:
```json
{
  "operation": "upload",
  "propertyName": "data",
  // 기타 설정
}
```

### 인사이트
파일명이 'undefined'로 저장되는 문제는 'Property Name' 설정이 이전 노드에서 바이너리 데이터를 지정한 이름과 일치하지 않아서 발생했습니다. 이 설정을 명시적으로 지정하지 않으면, 파일명의 불일치 문제가 발생할 수 있습니다. 따라서 강의나 프로젝트에서 '바이너리 프로퍼티 이름 일치'의 중요성을 강조하는 것이 필요합니다.
