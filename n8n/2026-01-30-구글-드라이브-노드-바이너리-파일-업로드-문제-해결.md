# 구글 드라이브 노드: 바이너리 파일 업로드 문제 해결

## Context
n8n 워크플로우를 사용하는 과정에서 구글 드라이브 노드를 통해 바이너리 파일을 업로드할 때 파일명이 `undefined`로 업로드되는 문제가 발생했습니다. 이는 파일의 바이너리 데이터는 전송되지만 파일 이름이 올바르게 설정되지 않아 발생한 것입니다.

## Core
문제의 원인은 노드 설정에서 `Property Name`이 이전 노드에서 정의한 바이너리 키값과 일치하지 않아서 발생했습니다. 이를 해결하기 위해 다음과 같이 설정해야 합니다:

```plaintext
1. 이전 노드에서 바이너리 데이터를 준비할 때, 특정 키값(예: `data`)에 저장합니다.
2. 구글 드라이브 노드의 설정에서 `Property Name`을 해당 키값(`data`)과 정확히 일치시킵니다.
```

이 설정을 명시적으로 지정하지 않을 경우, 파일 내용은 업로드되더라도 파일 이름이 `undefined`로 표시됩니다.

## Insight
이 문제는 n8n을 활용하여 워크플로우를 설계할 때 바이너리 프로퍼티 이름의 일치 여부가 얼마나 중요한지를 보여줍니다. 워크플로우를 설계할 때 발생할 수 있는 잠재적 오류를 방지하기 위해, 바이너리 데이터를 다루는 노드 설정에서 `Property Name`을 정확하게 매칭하는 것이 필수적입니다. 이 점을 강조하여 강의를 진행할 계획입니다.
