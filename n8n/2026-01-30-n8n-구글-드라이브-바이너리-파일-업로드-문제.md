### Context
n8n에서 Google Drive 노드를 사용하여 바이너리 파일을 업로드할 때 파일명이 'undefined'로 저장되는 문제가 발생했다. 이는 파일명 설정을 위한 'Property Name'이 이전 노드에서 정의한 바이너리 키값과 일치하지 않아서 발생한 문제였다.

### Core
```plaintext
// Google Drive 노드 설정 예시
{
  "name": "새로운 Google Drive 노드",
  "type": "n8n-nodes-base.googleDrive",
  "parameters": {
    "operation": "upload",
    "propertyName": "data" // 이전 노드의 바이너리 키값과 일치해야 함
  }
}
```

### Insight
Google Drive 노드에서 파일 업로드 시, 'Property Name'이 이전 노드의 바이너리 데이터 키와 정확히 일치하지 않으면 파일 이름이 'undefined'로 저장되는 문제가 있다. 따라서 노드 설정에서 이 값을 명시적으로 정의해야 한다. 특히 강의를 할 때 이러한 부분의 중요성을 강조해야 하며, 이는 데이터 흐름의 일관성을 유지하는 데 필수적이다.
