# Google Drive Node에서 바이너리 파일명 설정 문제 해결

## Context
n8n에서 Google Drive 노드를 사용하여 파일을 업로드할 때, 파일명이 'undefined'로 저장되는 문제를 발견했다. 이는 파일 이름이 전 노드에서 정의된 바이너리 키값과 일치하지 않아서 발생한 것이었다.

## Core
Google Drive 노드에서 파일 업로드 시, 파일명을 올바르게 지정하려면 'Property Name' 필드를 전 노드에서 정의한 바이너리 키값과 정확히 일치시켜야 한다. 아래는 문제를 해결하기 위한 설정 방법이다:

```json
{
  "nodes": [
    {
      "parameters": {
        "binaryPropertyName": "data"
      },
      "name": "Google Drive",
      "type": "n8n-nodes-base.googleDrive",
      "typeVersion": 1,
      "position": [
        500,
        300
      ]
    }
  ]
}
```

## Insight
이 설정을 통해 파일의 내용 뿐만 아니라 이름도 올바르게 업로드할 수 있었다. 특히, 'Property Name'이 전 노드의 바이너리 키값과 일치해야 한다는 점을 명확히 인지하는 것이 중요하며, 이를 강의나 문서에 강조하여 실수를 방지할 수 있다. 바이너리 프로퍼티 이름 일치의 중요성을 이해하면 작업의 정확성을 높일 수 있다.
