### Context
n8n의 Google Drive 노드에서 바이너리 파일을 업로드할 때, 파일명이 'undefined'로 저장되는 문제가 발생했습니다. 이는 노드 설정에서 'Property Name'을 전 노드에서 정의한 바이너리 키 값과 정확히 일치시키지 않아서 생긴 문제였습니다.

### Core
```javascript
{
  "nodes": [
    {
      "parameters": {
        "resource": "file",
        "operation": "upload",
        "binaryPropertyName": "data" // 전 노드에서 정의한 키 값
      },
      "type": "n8n-nodes-base.googleDrive",
      ...
    }
  ]
}
```

### Insight
Google Drive 노드에서 파일을 업로드할 때, 바이너리 데이터의 ‘Property Name’을 명시적으로 지정해야 합니다. 이를 전 노드에서 설정한 바이너리 키와 일치시켜야 파일 이름이 올바르게 저장됩니다. 미지정 시 파일의 내용은 전송되지만 이름이 'undefined'로 저장될 위험이 있습니다. 이 설정은 n8n 워크플로우의 안정성을 높이는 중요한 포인트입니다.
