# 바이너리 파일 업로드 시 파일명 문제 해결

## Context
오늘 n8n에서 Google Drive 노드를 이용하여 바이너리 파일을 업로드하는 과정에서 파일명이 'undefined'로 저장되는 문제가 발생했다. 이는 바이너리 프로퍼티 설정에서 문제가 있다는 것을 발견했다.

## Core
Google Drive 노드 설정 시, 바이너리 파일의 `Property Name`을 이전 노드에서 정의한 바이너리 데이터의 키 값(예: `data`)과 정확히 일치시켜야 한다. 이를 일치시키지 않으면 파일 내용은 전송되지만 파일명이 깨진다.

```markdown
1. Google Drive 노드에서 'File Name'을 설정할 때, 반드시 이전 노드에서 정의한 바이너리 키 값과 동일한 'Property Name'을 입력.
2. 예시: 이전 노드의 출력 데이터가 `{ "data": { ... } }` 라면, Google Drive 노드의 'Property Name'에 `data`를 입력한다.
```

## Insight
바이너리 파일 업로드 시 프로퍼티 이름이 일치하지 않으면, 데이터는 업로드되더라도 파일명이 'undefined'로 처리된다. 이러한 문제는 n8n의 Google Drive 노드뿐만 아니라 다른 바이너리 파일을 다루는 노드에서도 동일하게 발생할 수 있으므로 주의가 필요하다. 강의나 실습 시 이 점을 명확히 설명하여, 더 나은 이해와 활용을 도모할 것이다.
