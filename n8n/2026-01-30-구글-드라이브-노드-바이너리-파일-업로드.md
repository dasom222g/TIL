# Google Drive 노드에서 바이너리 파일 업로드 문제 해결

## Context
n8n을 사용하여 Google Drive에 바이너리 파일을 업로드할 때, 파일명이 'undefined'로 저장되는 문제가 발생했습니다. 이는 노드 설정에서 'Property Name'이 전 노드에서 정의한 바이너리 키값과 일치하지 않았기 때문입니다.

## Core
```plaintext
// Google Drive Node Settings
{
  "operation": "uploadFile",
  "resource": "file",
  "binaryPropertyName": "data", // 이 값이 전 노드의 키값과 일치해야 합니다
  // 기타 설정 값들
}
```

## Insight
이 설정을 명시적으로 맞춰주지 않으면, 파일의 내용은 제대로 업로드되지만 파일명이 'undefined'로 저장됩니다. 이를 해결하기 위해 전 노드의 바이너리 키값과 Google Drive 노드의 'Property Name'을 일치시켜야 합니다. 이 원리를 이해하고 적용함으로써 보다 안정적인 워크플로우 설계가 가능합니다. 향후 강의시, 바이너리 프로퍼티 이름을 일치시키는 것이 얼마나 중요한지 강조할 예정입니다.
