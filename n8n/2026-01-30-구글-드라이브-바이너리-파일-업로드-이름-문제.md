### Context
구글 드라이브 노드를 사용하여 바이너리 파일을 업로드하는 과정에서 파일명이 'undefined'로 들어가는 문제가 발생했습니다. 이 문제는 노드 설정에서 'Property Name'의 설정이 올바르지 않아서 발생했습니다.

### Core
구글 드라이브 노드의 설정에서 'Property Name'을 전 노드에서 정의한 바이너리 키 값과 정확히 일치시켜야 합니다. 예를 들어, 이전 노드에서 바이너리 데이터를 'data'라는 키로 설정한 경우, 다음과 같이 설정해야 합니다:

```json
{
  "nodeType": "n8n-nodes-base.googleDrive",
  "parameters": {
    "fileBinaryPropertyName": "data"
  }
}
```

이 설정을 통해 파일 이름이 올바르게 정의되고, 'undefined' 문제를 해결할 수 있습니다.

### Insight
바이너리 파일 업로드 시에는 'Property Name'을 명시적으로 설정하는 것이 중요합니다. 설정하지 않으면 파일의 내용은 정상적으로 전송되지만, 파일 이름이 'undefined'로 저장될 수 있습니다. 이는 특히, 바이너리 데이터의 프로퍼티 이름이 일치하지 않는 경우 발생할 수 있는 문제입니다. 따라서, 강의나 문서에서 바이너리 프로퍼티 이름의 일치성을 강조해야 합니다.
