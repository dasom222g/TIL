# n8n 구글 드라이브 노드에서 바이너리 파일 업로드 문제 해결

## Context
n8n에서 구글 드라이브(Google Drive) 노드를 사용하여 파일을 업로드할 때, 바이너리 파일의 이름이 'undefined'로 업로드되는 문제가 발생했습니다. 이는 이전 노드에서 정의한 바이너리 키값과 구글 드라이브 노드 설정의 'Property Name'이 일치하지 않아서 발생한 문제입니다.

## Core
구글 드라이브 노드에서 파일 업로드 시 정확한 바이너리 프로퍼티 이름을 지정하는 방법은 다음과 같습니다:

```json
{
  "parameters": {
    "resource": "file",
    "operation": "upload",
    "binaryPropertyName": "data"
  }
}
```
여기서 `binaryPropertyName`은 이전 노드에서 정의한 바이너리 데이터의 키 값과 일치해야 합니다. 예를 들어, 파일을 이전 노드에서 'data'라는 키로 정의했다면, 구글 드라이브 노드에서도 'data'로 설정해야 파일명이 정상적으로 전송됩니다.

## Insight
이 문제는 '바이너리 프로퍼티 이름 일치'의 중요성을 보여줍니다. 제대로 설정하지 않으면, 파일 자체는 정상적으로 전송되지만 이름이 'undefined'로 지정되어 불편함을 초래할 수 있습니다. 강의나 문서 작성 시 이 부분을 강조하여 배울 수 있도록 해야 합니다. 이는 특히 다양한 노드 간 데이터 전송 시 발생할 수 있는 문제들을 사전에 방지하는 데 중요한 역할을 할 것입니다.
