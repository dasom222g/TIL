### Context
n8n으로 첫 번째 자동화 워크플로우를 만들어 보았습니다. 디스코드에서 메시지를 수신하면 이를 트리거로 사용하고, 이를 통해 LLM 에이전트가 실행됩니다. 에이전트는 메시지를 TIL(오늘 배운 것) 형식으로 변환하고, 변환된 코드를 GitHub에 푸시합니다. 푸시가 완료되면 README.md 파일에 인덱스가 추가되고, 마지막으로 완료 메시지가 발신됩니다.

### Core
```javascript
// Discord에서 메시지 수신
{
  "name": "Discord Trigger",
  "type": "webhook",
  "parameters": {
    "path": "webhook-path",
    "events": ["MESSAGE_CREATE"]
  }
}

// LLM 에이전트 실행
{
  "name": "Run Agent",
  "type": "httpRequest",
  "parameters": {
    "url": "agent-endpoint",
    "method": "POST",
    "body": {/* 메시지 내용 */}
  }
}

// GitHub에 푸시
{
  "name": "GitHub Push",
  "type": "httpRequest",
  "parameters": {
    "url": "https://api.github.com/repos/your-repo/contents",
    "method": "PUT",
    "headers": {/* 인증 정보 */},
    "body": {
      "path": "파일경로",
      "message": "add TIL",
      "content": "base64-encoded-content"
    }
  }
}

// README.md 업데이트
{
  "name": "Update README",
  "type": "httpRequest",
  "parameters": {
    "url": "https://api.github.com/repos/your-repo/contents/README.md",
    "method": "PUT",
    "headers": {/* 인증 정보 */},
    "body": {
      "content": "base64-encoded-content"
    }
  }
}

// 완료 메시지 발신
{
  "name": "Send Completion Message",
  "type": "webhookReply",
  "parameters": {/* 메시지 설정 */}
}
```

### Insight
n8n을 처음 사용해 활동 자동화 워크플로우를 만드는데 성공했습니다. 노드의 입력과 출력만 이해하면 워크플로우 설계가 어렵지 않음을 깨달았습니다. 이 기회를 주신 후츠릿님께 감사를 표합니다.
